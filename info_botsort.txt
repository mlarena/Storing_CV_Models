tracker_type: botsort: Указывает тип трекера. 
В данном случае установлено значение botsort, но также может быть установлено значение bytetrack (другой вариант алгоритма SORT).

track_high_thresh: 0.5: Порог для первой ассоциации. 
Это пороговое значение уверенности для сопоставления обнаружений с существующими треками на первом этапе ассоциации.

track_low_thresh: 0.1: Порог для второй ассоциации. 
Это пороговое значение уверенности для сопоставления обнаружений с существующими треками на втором этапе ассоциации, 
которое обычно ниже первого порога, чтобы позволить более мягкое сопоставление.

new_track_thresh: 0.6: Порог для инициализации нового трека. 
Если обнаружение не соответствует ни одному из существующих треков с уверенностью выше этого порога, инициализируется новый трек.

track_buffer: 50: Размер буфера для расчета времени удаления треков. 
Этот параметр определяет, как долго трек должен храниться в памяти перед удалением, если он не сопоставлен ни с одним из обнаружений.

match_thresh: 0.8: Порог для сопоставления треков. 
Это пороговое значение IoU (Intersection over Union), используемое для определения, соответствует ли обнаружение существующему треку.

fuse_score: True: Следует ли объединять оценки уверенности с расстояниями IoU перед сопоставлением. Э
то может помочь улучшить точность процесса сопоставления, учитывая как оценки уверенности обнаружений, так и их пространственное перекрытие.
